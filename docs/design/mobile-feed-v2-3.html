Surface 01 · Native Mobile

Armal News — mobile feed, end to end

React Native app — no iOS/Android UI differentiation. Light and dark for every screen. 375×812 viewport throughout. Share sheet is OS-rendered via expo-sharing — no custom Armal UI needed.

Light Dark
01

Feed home

Default landing · vertical snap

One full-screen card per swipe. Image covers top half, headline card straddles the image/surface boundary, summary sits centered in the lower half. Category icon top-left, wordmark top-center. "Tap to read →" bottom-right.

02

Feed — active category filter

Pill dismiss · 44×44 hit target

When a category is active: wordmark anchors the top of the image, filter pill sits directly below it — both centered, always readable against the image scrim. Tap ✕ to clear and return to all stories.

03

Category bottom sheet

~70% screen height · drag handle

Tapping the category area brings up a bottom sheet — rounded top corners, drag handle, dimmed backdrop. Each row is 56px, icon + name + active checkmark. "All stories" clears any active filter.

04

Deep-dive article

Pushed view · edge-swipe to dismiss

Light shows initial state: hero, glassy back button, eyebrow + title + italic summary. Dark shows scrolled state: collapsed hero, pull quote, tags, accent source CTA.

05

First-launch onboarding

Single screen · no account · no category picker

No tour, no sign-up, no carousel. One screen, one promise, one button. Tap Continue and you're in the feed.

06

App icon · adaptive variants

iOS rounded-square mask · 1024×1024

Stylized "A" in italic Newsreader on warm-paper. Light and dark variants. No gradient, no badge.

07

Splash screen

Cold-launch · ~600ms

Wordmark centered on background. No spinner, no progress bar — cold-launch is fast enough that decoration feels apologetic.
